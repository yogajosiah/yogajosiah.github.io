概述

单阶段： 直接用CNN（卷积神经网络）求特征回归，确定x,y,w,h进而确定要识别物体的位置和范围

模型评估的相关指标

1. 模型对单个图像进行目标识别和物体检测

  • IOU: 预测边界框与真实边界框的重合程度。

$$(预测值∩真值)/(预测值∪真值)$$

  • 置信度： 表示模型对预测结果的置信程度，通常是一个分数或一个概率值。
IOU/置信度 置信度高 置信度低
IOU高 模型很自信 模型不自信且存在一定不确定性
IOU低 模型存在一定误判却过度自信 ——————

2. 模型对整个数据集和一组图片进行目标识别和物体检测

  • 精度(precition)： 衡量模型的准确性，预测的正类里面样本中有多少位真正的正类。$$TP/TP+FP$$
  • 召回率(recall)： 正确检测出的正类样本占所有正类的比例。$$TP/TP+FN$$
TP FP TN FN
检测正确的正类 检测错误的正类 检测正确的负类 检测错误的负类
正确的检测正确 错误的检测为正确 错误的检测为错误 正确的检测为错误

正类： 目标检测和物体识别中的包含待检测目标的样本
负类： 目标检测和物体识别中的不包含待检测目标的样本
TP+FP= 预测为正类的数量(置信度>阈值的样本数量)
TP+FN= 真是正类的数量(存在待检测目标的所有的样本数量)

3. 对模型评估的综合考量指标：map

4. NMS非极大值抑制：评估的时候会遇到。

核心思想

1.单次前向传播： 与传统目标检测算法（如基于区域的方法）不同，YOLOv1将目标检测任务简化为一个单次前向传播的问题。它通过在图像上应用网格划分，将图像划分为多个区域，并在每个区域上进行预测。

2.网格划分： YOLOv1将输入图像划分为固定大小的网格（S*S=7x7），每个网格负责预测图像中是否存在目标以及目标的位置。每个网格预测的目标由预定义的边界框数量（B=2）和类别数确定。

3.边界框预测： 每个网格通过预测边界框的坐标和相关性来定位目标。每个边界框包含一组位置和大小参数（x,y,w,h）。通过对这些参数进行回归预测，可以得到边界框的位置。

4.目标类别预测： 除了边界框预测，每个网格还负责预测目标所属的类别。通过在每个网格上使用多类别的Softmax分类器，可以对目标进行分类(20)。

5.损失函数： YOLOv1使用一个综合的损失函数来同时优化边界框的位置预测和类别预测。损失函数考虑了边界框的位置误差、类别预测的置信度和类别预测的准确性，以综合评估模型的性能。

博客图床

PicGo+Github 搭建图床

@[TOC]

Github 创建仓库

1. 在Github上创建一个新的仓库，填写仓库信息。

2. 打开设置生成一个Token

  • 进入settings页面

  • 在该页面左侧的最下面找到Developer settings

  • 选择左侧的Personal access tokens,我选择的是经典款，点击Generate new token

  • 填写note，勾选repo，点击生成按钮Generate token,将生成的token复制保存，不然就需要重新生成。

PicGo配置

1. 安装PicGo

我的下载地址PicGo-Setup-2.4.0-beta.3-x64.exe

2. 配置PicGo

图床配置名： 不用管；
设置仓库名： [Github用户名]/[第一步新建的仓库名]；
分支： main；
设定token: 把刚刚复制保存到的tocken粘贴过来；
指定路径： 选填，如果填写，则会存储在此路径下；
设定自定义域名： 我没设置，PicGo帮我自动生成了。

保存你的设置，shift+win+R截屏之后，按快捷键shift+ctrl+P就可以自动上传并生成URL

遇到的问题

1.上传失败
可以查看日志来判断具体是哪里出问题了
“C:\Users\ge200\AppData\Roaming\picgo\picgo.log”

2.上传成功但是实现的是图裂
可能的原因：

  • 仓库私有:去仓库设置里里面改成public
  • 域名不正确：如果没有自己的域名，不如下面的自定义域名就空着。

ps:想要删库在仓库setting的最下面了，整个操作过程都需要代理服务器的帮助

图像识别小车(电赛速成篇)

[TOC]

一、硬件准备：

  • 使用STM32F103作为控制板，连接电机驱动模块以控制小车的移动。
  • 连接摄像头模块到Jetson Nano，用于获取实时图像。
  • 将STM32F103和Jetson Nano连接起来，可以通过串口通信或其他适当的通信方式进行数据传输。

二、图像采集与处理：

  • 在Jetson Nano上，使用适当的库（如OpenCV）初始化摄像头模块，并获取实时图像。
  • 对图像进行预处理，如去噪、颜色空间转换、边缘检测等，以突出红线并提取特征。
  • 运行图像处理算法（如色彩分割、轮廓检测、直线拟合等）来识别红线的位置和方向。

三、红线跟踪与控制：

  • 将处理后的图像数据传输到STM32F103上进行处理。
  • 根据红线的位置和方向信息，控制电机驱动模块，使小车沿着红线行驶。
  • 使用PID控制算法来实现平稳的跟踪和控制。

四、实时性与优化：

  • 考虑使用优化技术和算法，以提高红线识别和控制的实时性。
  • 对图像处理和电机控制的算法进行性能优化，以确保在给定的计算资源下实现实时响应和较低的延迟。

五、调试与测试：

  • 在实现过程中，进行逐步调试和测试，确保硬件和软件的正常运行。
  • 可以使用调试工具和日志打印来检查数据传输、图像处理和电机控制的正确性。
